# 讨论全记录：易经-贝叶斯世界模型

## 思想的起源

### 对当前 AI 范式的批判

当前最先进的大语言模型（GPT、Claude、DeepSeek 等）的核心机制是**自回归下一个 token 预测**。它们本质上在学习"给定前面的词，下一个词最可能是什么"——即人类语言的统计分布。

这里有一个根本问题：**它们学的是"人类如何说"，而非"世界如何是"。**

一个完美的语言模型可以写出符合语法的"太阳从西边升起"——因为它只关心 token 概率，不关心物理世界的真实约束。

### 构想的雏形

如果我们要构建一个真正理解世界的 AI，它需要：
1. **一个世界状态空间**：对"世界可能处于什么状态"的系统划分
2. **一个更新机制**：当观察到新事实时，如何修正对世界的认知
3. **可解释性**：能回溯"为什么做出这个预测"

这时，《易经》进入视野。

### 为什么是《易经》？

《易经》不是算命工具。它是一套**对系统状态进行划分和推演的数学模型**：

| 易经概念 | 现代 AI 对应 |
|---------|------------|
| 64 卦 | 潜变量状态空间 (64 个离散状态) |
| 八卦 (三爻) | 状态空间的基本维度 (8 个基础模式) |
| 爻变 | 状态转移规则 |
| 吉凶 | 反馈信号 (reward / evidence) |
| 卦辞 | 对状态-因果关系的自然语言描述 |

64 卦 = $2^6$ 个状态。这只是周文王时代的算力限制——用 6 个二进制位（爻）编码世界状态。现代算力可以实现近乎无限的"分辨率"。

**关键洞察**：易经提供了一个**结构化的先验因果框架**——一个"世界的脊梁骨"。贝叶斯更新提供了"血肉"——用每一次真实反馈修正这个框架。

### 贝叶斯更新为核心

```
先验分布 P(世界状态)
    ↓
观测到证据 (天气实际变化)
    ↓
贝叶斯定理: P(状态|证据) ∝ P(证据|状态) × P(状态)
    ↓
后验分布 → 成为新的先验
    ↓
确定性升级！
```

这解决了当前 LLM 的四个核心问题：
1. **样本效率**：结构化先验 → 不需要互联网级别数据
2. **可解释性**：灰箱 → 可回溯因果链（哪个卦象主导了这次预测）
3. **遗忘问题**：定向局部更新 → 新证据只修正相关卦象
4. **不确定性量化**：Dirichlet 分布天然表达不确定性

## MVP 开发历程

### V1：初始构想

**设计**：64 卦每个维护一个 8 维 Dirichlet 分布（预测独立天气类型）。

**问题**：模型预测的是"什么天气会出现"，而非"给定当前天气，下一个天气是什么"。没有建模转移动态。

**结论**：需要卦象特定的**条件转移矩阵**。

### V2：卦象转移矩阵

**设计**：每个卦象 $h$ 维护 $8 \times 8$ Dirichlet 转移矩阵：
$$p(w_{t+1} | w_t, h) \sim \text{Dirichlet}(\alpha_h[w_t, :])$$

总参数：$64 \times 8 \times 8 = 4096$ 个 Dirichlet 参数。

**结果**：
- 易经贝叶斯达到 35% 准确率（vs 随机 12.5%，NN 23%）
- 但仅在单次运行上测试，缺乏统计显著性
- 传统先验 vs 统一先验无明显优势

**问题**：单次运行不可靠；传统先验未体现优势。

### V3：严格验证实验

**改进**：
1. **先验消融**：传统 vs 统一 vs 随机（独立 Dirichlet 而非排列）
2. **数据效率曲线**：100→3000 天训练量
3. **5 种子 + 95% 置信区间**：排除运气
4. **训练/评估分离**：评估时不更新模型
5. **多机制天气**：4 个 regime 的非平稳 Markov（论证 64 卦容量需求）

**发现的关键 bug**：IChing-Scrambled（排列先验）与 IChing-Bayes 数学等价——因为模型对 64 个卦象做无差别加权聚合，排列不改变加权和。修复为独立随机 Dirichlet 先验。

**V3 最终结果**：

| Days | IChing-Bayes | IChing-Uniform | IChing-Random | Neural-Net |
|------|-------------|---------------|--------------|-----------|
| 100  | 12.1% | 25.5% | 10.5% | 10.1% |
| 200  | 4.9%  | 26.1% | 11.2% | 13.2% |
| 500  | 22.7% | 50.7% | 12.6% | 12.1% |
| 1000 | 13.9% | 48.3% | 17.9% | 28.3% |
| 2000 | 40.1% | 46.7% | 43.3% | 37.5% |
| 3000 | 46.8% | 47.5% | 45.7% | 41.8% |

## 核心洞察与哲学反思

### 1. 结构化的价值

传统卦象先验始终优于独立随机先验（全数据区间）。**卦象之间的八卦共享关系**提供了有效的归纳偏置——这是"无结构"的随机先验所不具备的。

### 2. 先验与数据的辩证关系

Uniform 先验在小数据区间优于传统先验。这完全符合贝叶斯理论：**当先验"错误"时，无信息先验更安全。但随着证据积累，结构化先验逐步兑现价值。**

这恰好证明了贝叶斯框架的自我修正能力——即使从"不完美"的先验出发，足够的数据也能将模型拉向真理。

### 3. 从 64 到 ∞

64 卦是周文王时代的算力极限。如果我们将 6 个爻扩展到 $n$ 个爻，状态空间变为 $2^n$——现代算力下 $n$ 可以非常大。

这暗示了一个更深层的猜想：**世界本身或许就是一个高维潜变量空间，易经只是对它的一次低维投影。** 64 卦之所以"灵验"，不是因为 64 这个数字神秘，而是因为它捕获了真实世界状态空间的某种低维结构。

### 4. "五维空间"猜想

在讨论中产生了一个更大胆的猜想：

> 如果世界是一个高维连续状态流形，易经的 64 卦是对这个流形的一次离散采样。那么真正的"世界模型"应该在这个连续流形上进行贝叶斯推理——用现代算力实现"无限分辨率的卦象"。

这类似于从词袋模型到 word2vec 再到 GPT 的演进：64 卦是"词袋"，八卦是"embedding 维度"，真正的世界模型需要学习连续的潜变量表示。
